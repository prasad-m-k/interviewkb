## Suggested starting sources
Drop any of these into `sources/` and ask for an ingest:
- *Cracking the Coding Interview* (Gayle Laakmann McDowell) — foundational DSA
- [The System Design Primer](https://github.com/donnemartin/system-design-primer) — system design survey
- *Designing Data-Intensive Applications* (Martin Kleppmann) — deep system design
- NeetCode 150 problem list — curated problem set
- Target company interview guides (Glassdoor, Blind, LeetCode discuss)

## How to use this wiki

**Add a source:** Drop a file in `sources/` (or paste content) and say "ingest this".

**Ask a question:** Ask anything — "what are the main DP patterns?", "how do I prepare for Google system design?". The answer may get filed as a new wiki page.

**Health check:** Say "lint the wiki" for a gap analysis and suggested next steps.
